## 1. BPE 토크나이저

**파일**: [`tokenizer/train_tokenizer.py`](../tokenizer/train_tokenizer.py)

### 역할

텍스트 문자열을 모델이 처리할 수 있는 숫자 배열로 변환합니다.  
모델은 문자를 직접 이해하지 못하므로, 모든 텍스트는 반드시 숫자로 변환되어야 합니다.

**BPE(Byte-Pair Encoding)** 는 자주 함께 등장하는 문자 쌍을 반복적으로 합쳐 어휘 사전을 만드는 방식입니다.

```
학습 과정 예시:
  처음: ["h", "e", "l", "l", "o"]
  1회: "l"+"l" 자주 등장 → "ll" 합침 → ["h", "e", "ll", "o"]
  2회: "he" 자주 등장 → ["he", "ll", "o"]
  ...반복...
  최종 사전: {"hello": 4231, "world": 892, ...} 총 32,000개
```

처음 본 단어도 서브워드로 분해해 처리할 수 있습니다.

```
"transformer" → ["trans", "former"]  (사전에 없어도 처리 가능)
"트랜스포머"  → ["트랜", "스포", "머"]
```

**대안**: WordPiece(BERT 사용), SentencePiece(LLaMA 사용), 문자 단위 토크나이저. BPE는 GPT 계열의 표준입니다.

## 2. 패딩 & ignore_index

**파일**: [`tokenizer/train_tokenizer.py:79`](../tokenizer/train_tokenizer.py), [`model/dense_transformer.py:93`](../model/dense_transformer.py)

### 역할

GPU는 배치 안의 모든 문장을 동시에 처리하므로, 길이가 다른 문장들을 **같은 길이로 맞춰야** 합니다.  
짧은 문장 뒤에 `<pad>` 토큰을 채워 길이를 맞추는 것을 패딩이라 합니다.

```
문장1: ["안녕", "하세요"]            → [1423, 892, <pad>, <pad>]
문장2: ["오늘", "날씨", "가", "좋다"] → [231,  44,   892,  1201]
```

그런데 `<pad>`는 실제 의미 없는 자리 채우기이므로, 손실 계산 시 무시해야 합니다.  
`ignore_index=-100`이 그 역할을 합니다. 패딩 위치의 레이블을 -100으로 표시하면 Cross-Entropy가 해당 위치를 건너뜁니다.

**대안**: 동적 패딩(배치 내 최대 길이에만 맞춤), 버킷 샘플링(비슷한 길이끼리 묶기). 이 프로젝트는 고정 `block_size`로 잘라 패딩을 최소화합니다.

## 3. 토큰 임베딩 (Untied)

**파일**: [`model/dense_transformer.py:36`](../model/dense_transformer.py), [`model/dense_transformer.py:51`](../model/dense_transformer.py)

### 역할

숫자(토큰 ID)를 의미를 담은 벡터로 변환합니다.  
32,000개 단어 각각을 768차원 벡터로 매핑하는 거대한 조회 테이블입니다.

```
"고양이" → 토큰 ID 1423 → [0.31, -0.12, 0.87, ...(768개 숫자)]
"강아지" → 토큰 ID 892  → [0.29, -0.09, 0.81, ...(768개 숫자)]
```

학습이 진행될수록 의미가 비슷한 단어의 벡터는 가까워지고, 다른 단어는 멀어집니다.

**Untied 구조**: 입력 임베딩(`token_embeddings`)과 출력 분류기(`lm_head`)의 가중치를 **별도로** 유지합니다.  
일부 모델(GPT-2 등)은 이 두 행렬을 공유(Tied)해 파라미터를 절약합니다. 이 프로젝트는 Untied를 선택해 표현력을 높였습니다.

**대안**: Tied Embedding(파라미터 절약), 문자 단위 임베딩. 현대 대형 모델은 대부분 Untied를 사용합니다.

## 4. RoPE (Rotary Position Embedding)

**파일**: [`model/rope.py`](../model/rope.py), [`model/attention.py:67-68`](../model/attention.py)

### 역할

Attention은 기본적으로 단어 순서를 알지 못합니다. "고양이가 쥐를 잡았다"와 "쥐가 고양이를 잡았다"를 동일하게 처리합니다.  
위치 정보를 주입해 모델이 단어 순서를 인식하게 만드는 것이 포지셔널 인코딩입니다.

RoPE는 Query와 Key 벡터를 위치에 따라 **회전**시키는 방식입니다.

```
위치 0의 "고양이" 벡터: [0.3, 0.8, ...]  → 0도 회전
위치 1의 "고양이" 벡터: [0.3, 0.8, ...]  → 5도 회전
위치 2의 "고양이" 벡터: [0.3, 0.8, ...]  → 10도 회전
```

같은 단어라도 위치가 다르면 다른 방향을 가리키게 되어, Attention이 위치 차이를 감지할 수 있습니다.  
특히 두 단어 사이의 **상대적 거리**를 자연스럽게 인코딩하는 장점이 있습니다.

**대안**: 사인/코사인 Positional Encoding(원본 Transformer), 학습 가능한 위치 임베딩(GPT-2), ALiBi. RoPE는 LLaMA, Mistral, Gemma 등 현대 LLM의 표준입니다.

## 5. Multi-Head Attention

**파일**: [`model/attention.py`](../model/attention.py)

### 역할

문장에서 단어들이 서로를 얼마나 참조해야 하는지 계산하는 핵심 메커니즘입니다.

**단일 헤드 Attention** 이면 하나의 관점에서만 관계를 파악합니다.  
**Multi-Head** 는 8개의 서로 다른 관점을 동시에 적용합니다.

```
입력 768차원 → 8개 헤드로 분할 → 각 헤드는 96차원(768/8)에서 독립적으로 Attention 계산

헤드 1: 문법적 관계 (주어-동사)를 포착할 수 있음
헤드 2: 의미적 관계 (대명사-지시 대상)를 포착할 수 있음
...
헤드 8: 또 다른 패턴

→ 8개 결과를 이어붙여(concat) 다시 768차원으로 합침
```

각 헤드가 무엇을 포착하는지는 학습 중 자동으로 결정됩니다.

**Attention 계산 공식**:
```
scores = Q @ Kᵀ / √96   (유사도 측정 + 스케일링)
probs  = softmax(scores) (확률 분포로 변환)
output = probs @ V       (가중 평균으로 정보 취합)
```

**대안**: Multi-Query Attention(K·V를 헤드 간 공유, 추론 속도 향상), Grouped-Query Attention(LLaMA2·3 사용). 이 프로젝트는 표준 MHA를 사용합니다.

## 6. Causal Masking

**파일**: [`model/attention.py:38-42`](../model/attention.py)

### 역할

디코더 전용 모델은 텍스트를 **왼쪽에서 오른쪽으로** 한 단어씩 생성합니다.  
따라서 현재 단어를 예측할 때 미래 단어를 참조해서는 안 됩니다. 이를 강제하는 것이 Causal Masking입니다.

```
"오늘 날씨가 좋다"를 학습할 때:

"오늘"을 처리할 때:    [오늘, ████, ████, ████]  → 자기 자신만 참조
"날씨가"를 처리할 때:  [오늘, 날씨가, ████, ████] → 앞 2개만 참조
"좋다"를 처리할 때:    [오늘, 날씨가, 좋다, ████] → 앞 3개만 참조
```

구현 방식: Attention 점수 행렬의 미래 위치에 `-inf`를 더합니다. Softmax를 거치면 `-inf`는 0이 되어 해당 위치 참조 확률이 완전히 0이 됩니다.

**대안**: BERT처럼 마스킹 없이 양방향 참조 허용(분류 태스크에 적합). 생성 모델은 반드시 Causal Mask가 필요합니다.

## 7. SwiGLU FFN

**파일**: [`model/ffn.py`](../model/ffn.py)

### 역할

Attention이 "어떤 단어를 참조할까"를 결정한다면, FFN은 그 결과를 가지고 **비선형 변환**을 수행합니다.  
각 위치(단어)를 독립적으로 처리하며, 모델의 "지식 저장소" 역할을 한다고 알려져 있습니다.

**SwiGLU 구조**:
```
입력 x (768차원)
  ├→ W1 → 3072차원 → SiLU 활성화 함수 적용 (게이트 경로)
  └→ W2 → 3072차원 (값 경로)

게이트 × 값 → 원소별 곱셈 → W3 → 768차원 출력
```

게이트 경로가 값 경로의 각 차원을 얼마나 "통과시킬지" 조절합니다. 불필요한 정보를 억제하는 효과가 있습니다.

**대안**: ReLU FFN(원본 Transformer), GELU FFN(GPT-2, BERT). SwiGLU는 LLaMA, PaLM, GPT-4 등 현대 대형 모델의 표준입니다. 동일한 파라미터 수에서 Perplexity를 약 1~2% 낮추는 것으로 알려져 있습니다.

## 8. Pre-RMSNorm

**파일**: [`model/normalization.py`](../model/normalization.py), [`model/transformer_block.py:42`](../model/transformer_block.py)

### 역할

학습 중 벡터 값이 너무 크거나 작아지는 것을 방지합니다. 이를 방치하면 역전파 시 기울기가 폭발하거나 소멸합니다.

**RMSNorm 공식**:
```
RMSNorm(x) = x / sqrt(mean(x²)) × weight
```

벡터의 크기(RMS, 제곱평균제곱근)로 나눠 스케일을 정규화합니다.

**Pre-Norm**: 정규화를 Attention/FFN **이전**에 적용합니다.
```
x = x + Attention(RMSNorm(x))   ← 입력을 먼저 정규화한 뒤 Attention
x = x + FFN(RMSNorm(x))         ← 입력을 먼저 정규화한 뒤 FFN
```

반대로 원본 Transformer는 Post-Norm(연산 후 정규화)을 사용했는데, 깊은 레이어에서 학습이 불안정해지는 문제가 있습니다. Pre-Norm은 이 문제를 해결해 12층 이상의 깊은 모델에서도 안정적으로 학습됩니다.

**대안**: LayerNorm(BERT, 원본 Transformer — 평균을 빼는 추가 연산 포함), Post-Norm. RMSNorm은 LayerNorm 대비 연산 비용이 약 7~10% 적습니다.

## 9. 잔차 연결 (Residual Connection)

**파일**: [`model/transformer_block.py:42-46`](../model/transformer_block.py)

### 역할

각 레이어의 출력에 입력을 그대로 더하는 구조입니다.

```
일반 레이어:   output = Layer(x)
잔차 연결:     output = x + Layer(x)
```

12개 레이어를 쌓으면 역전파 시 기울기가 레이어를 거치며 점점 작아집니다(기울기 소실). 잔차 연결은 기울기가 더하기 경로로 직접 흐를 수 있어 이 문제를 해결합니다.

```
역전파 기울기 흐름:
잔차 없음: 레이어12 → 레이어11 → ... → 레이어1  (곱셈으로 소실)
잔차 있음: 레이어12 → (+) → 레이어11 → (+) → ... → 레이어1  (덧셈 경로로 보존)
```

또한 레이어가 아무것도 하지 않아도 되는 선택지(항등 함수)를 주어, 필요한 경우에만 변환을 적용할 수 있습니다.

**대안**: Highway Network(게이트로 잔차 비율 조절), DenseNet(모든 이전 레이어 연결). 잔차 연결은 ResNet(2015)에서 이미지 분야에서 처음 사용되어 Transformer에도 채택되었습니다.

## 10. Dropout

**파일**: [`model/attention.py:33`](../model/attention.py), [`model/ffn.py:26`](../model/ffn.py), [`model/config.py:14`](../model/config.py)

### 역할

학습 중 무작위로 일부 뉴런을 비활성화해 모델이 특정 패턴에 과도하게 의존하지 않도록 하는 정규화 기법입니다.

```
dropout=0.1 설정 시:

학습 중: 매 forward마다 10%의 값을 무작위로 0으로 만듦
  [0.3, 0.8, 0.2, 0.9, ...] → [0.3, 0.0, 0.2, 0.9, ...]

추론 중: 비활성화 없이 모든 값 그대로 사용
```

특정 뉴런 조합에만 의존하면 그것이 없는 상황을 처리 못하는 과적합이 발생합니다. Dropout은 모델이 여러 경로로 분산된 표현을 학습하도록 강제합니다.

**대안**: Weight Decay(가중치 크기 패널티), DropPath(레이어 단위 드롭), 데이터 증강. 최근 대형 모델들은 데이터가 충분히 많아 Dropout 없이도 과적합이 적다는 이유로 비율을 0으로 설정하는 경우도 있습니다.

## 11. Teacher Forcing & Causal Shift

**파일**: [`model/dense_transformer.py:87-97`](../model/dense_transformer.py)

### 역할

**Teacher Forcing**: 학습할 때 모델의 이전 예측 결과가 아닌, **정답 토큰**을 다음 단계의 입력으로 사용하는 방식입니다.

```
문장: "오늘 날씨가 좋다"

Teacher Forcing 없음 (자기회귀):
  "오늘" → 모델 예측 "날씨" → "날씨"를 입력으로 → 예측 "좋아" → ...
  (예측이 틀리면 오류가 누적됨)

Teacher Forcing 있음:
  "오늘" → 모델 예측 (맞든 틀리든) → 정답 "날씨"를 다음 입력으로 강제
  (항상 올바른 맥락에서 다음 단어 예측 학습)
```

**Causal Shift**: `labels=input_ids`를 그대로 전달하면 "현재 단어로 현재 단어를 예측"하는 문제가 생깁니다. 이를 한 칸 밀어서 "이전 단어들로 다음 단어를 예측"하도록 만드는 처리입니다.

```python
shift_logits = logits[..., :-1, :]   # 마지막 위치 제거: 예측
shift_labels = labels[..., 1:]       # 첫 위치 제거:    정답

# 결과: 위치 0의 예측으로 위치 1의 정답을 맞추는 학습
```

**대안**: Scheduled Sampling(학습 후반부에 점진적으로 모델 예측을 입력으로 사용). Teacher Forcing은 구현이 단순하고 학습이 안정적이어서 대부분의 언어 모델에서 사용됩니다.

## 12. Cross-Entropy Loss

**파일**: [`model/dense_transformer.py:93-97`](../model/dense_transformer.py)

### 역할

모델의 예측이 정답과 얼마나 다른지 수치로 측정합니다. 이 수치를 낮추는 방향으로 파라미터를 업데이트합니다.

```
정답: "좋다" (토큰 ID 1201)
모델 출력 확률 분포 (32,000개 단어):
  "좋다":  0.6  ← 정답
  "맑다":  0.2
  "나쁘다": 0.1
  ...

Cross-Entropy = -log(0.6) = 0.51  ← 이 값을 낮추는 방향으로 학습

만약 확률이 0.9라면: -log(0.9) = 0.105  (낮은 손실 = 잘 예측)
만약 확률이 0.1이라면: -log(0.1) = 2.30  (높은 손실 = 잘못 예측)
```

**대안**: Focal Loss(어려운 샘플에 더 집중), Label Smoothing(정답 확률을 1 대신 0.9로 두어 과신 방지). 언어 모델의 표준 손실 함수는 Cross-Entropy입니다.

## 13. AdamW 옵티마이저

**파일**: [`train/train.py`](../train/train.py)

### 역할

손실 함수의 기울기(역전파 결과)를 이용해 모델 파라미터를 업데이트하는 알고리즘입니다.

**Adam**: 각 파라미터마다 독립적인 학습률을 적용하고, 과거 기울기의 이동 평균을 이용해 업데이트 방향과 크기를 결정합니다.

```
기본 SGD:  param = param - lr × gradient
Adam:      param = param - lr × (gradient 이동 평균 / 기울기 변동성)
```

**AdamW**: Adam에 Weight Decay(L2 정규화)를 올바르게 적용한 버전입니다. 가중치가 너무 커지는 것을 방지해 일반화 성능을 높입니다.

**대안**: SGD + Momentum(단순하지만 학습률 조정이 어려움), Adafactor(메모리 효율적, T5에서 사용), Lion(구글 연구, 메모리 절약). AdamW는 거의 모든 현대 언어 모델의 표준 옵티마이저입니다.

## 14. Learning Rate Warmup

**파일**: [`train/train.py`](../train/train.py)

### 역할

학습 초반에 학습률을 0에서 목표값까지 서서히 올리는 전략입니다.

```
Warmup 없음:
  스텝 1: lr = 0.001  (큰 학습률로 시작 → 초기 파라미터가 무작위라 발산 위험)

Warmup 있음:
  스텝 1:    lr = 0.000001
  스텝 100:  lr = 0.0001
  스텝 1000: lr = 0.001  ← 목표 학습률 도달
  스텝 2000: lr = 0.0008 (이후 점진적으로 감소)
```

초기 파라미터는 무작위이므로 기울기가 매우 불안정합니다. 학습률이 크면 이 불안정한 기울기로 인해 파라미터가 극단값으로 이동할 수 있습니다. Warmup은 초기를 안정적으로 통과하게 해줍니다.

**대안**: Cosine Annealing(코사인 곡선으로 학습률 감소), Constant LR(고정), OneCycleLR. 대부분의 Transformer 학습에서 Warmup + 이후 감소 스케줄을 조합해 사용합니다.

## 15. Mixed Precision (BF16)

**파일**: [`train/train.py`](../train/train.py), CLAUDE.md

### 역할

모델 파라미터와 연산을 32비트(FP32) 대신 16비트(BF16) 부동소수점으로 수행해 메모리와 속도를 개선합니다.

```
FP32: 32비트 → 범위: ±3.4×10³⁸, 정밀도: 소수점 7자리
BF16: 16비트 → 범위: ±3.4×10³⁸, 정밀도: 소수점 2자리
```

BF16은 FP32와 동일한 **범위**를 유지하면서 크기를 절반으로 줄입니다. 덕분에 수치 오버플로우 없이 메모리 사용량과 연산 시간을 약 절반으로 줄일 수 있습니다.

```
162M 모델 기준:
  FP32: 파라미터 저장에 약 650MB
  BF16: 파라미터 저장에 약 325MB
```

로컬 학습(MPS/CPU)에서는 `mixed_precision="no"`(FP32)로, Colab A100에서는 `"bf16"`으로 전환합니다.

**대안**: FP16(BF16보다 범위가 좁아 오버플로우 위험), INT8 양자화(추론 전용). A100 이상 GPU에서는 BF16이 표준입니다.

## 16. Gradient Clipping

**파일**: [`train/train.py`](../train/train.py)

### 역할

역전파로 계산된 기울기의 크기가 특정 임계값을 넘으면 잘라내는(clip) 기법입니다.

```
max_norm = 1.0 설정 시:

기울기 벡터의 L2 노름이 0.5 → 그대로 사용
기울기 벡터의 L2 노름이 3.0 → 1.0 크기로 축소 후 사용
```

손실이 갑자기 크게 튀는 상황(Loss Spike)이 발생하면 기울기가 폭발적으로 커집니다. 이를 그대로 반영하면 파라미터가 극단값으로 이동해 학습이 망가집니다. Gradient Clipping은 이런 상황에서 안전망 역할을 합니다.

**대안**: Gradient Scaling(FP16 학습 시 작은 기울기 보존용), Gradient Penalty. 거의 모든 대형 모델 학습에서 기본으로 적용합니다.

## 17. Perplexity

**파일**: [`train/evaluate.py`](../train/evaluate.py)

### 역할

언어 모델이 텍스트를 얼마나 잘 예측하는지 나타내는 평가 지표입니다.

```
Perplexity = exp(Cross-Entropy Loss)
```

"모델이 다음 단어를 예측할 때 평균적으로 몇 개의 후보 중 고르는 것과 같은가"로 해석할 수 있습니다.

```
PPL = 1    → 완벽 예측 (불가능)
PPL = 10   → 평균적으로 10개 후보 중 하나를 고르는 수준
PPL = 100  → 100개 후보 중 하나를 고르는 수준 (학습 초기)
PPL = 1000 → 무작위 수준
```

학습이 진행될수록 PPL이 낮아집니다. GPT-2(117M)의 WikiText-103 기준 PPL은 약 29입니다.

**대안**: BLEU(번역 품질), ROUGE(요약 품질), BPC(비트/문자 단위). Perplexity는 별도 태스크 없이 학습 진행 상황을 모니터링하기에 가장 간단하고 직접적인 지표입니다.
